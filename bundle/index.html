@nrk/core-components

Getting started

core-components provides a simple and accessbile layer of functionality and basic styling - making often used components a breeze.

Use from NPM

- recommended 
npm install @nrk/core-components --save
import components from '@nrk/core-components' // all JS comonents...
import input from '@nrk/core-input'           // ...or individual JS component (recommended)
import Input from '@nrk/core-input/jsx'       // ...or individual JSX component (recommended)

Use with Require.js

 
<link rel="stylesheet" src="https://static.nrk.no/core-components/X.X.X/core-components.min.css"> <!-- Optional styling -->
<script>
require(['https://static.nrk.no/core-components/X.X.X/core-components.min.js'], function(components) { // all...
  /* use components.input etc. here */
});
</script>

Use with script tag

- good for prototyping, avoid in production 
<link rel="stylesheet" src="https://static.nrk.no/core-components/X.X.X/core-components.min.css"> <!-- Optional styling -->
<script src="https://static.nrk.no/core-components/X.X.X/core-components.min.js"></script>
<script>
  /* use window.components.input etc. here */
</script>

Dialog

<dialog> is an element with which the user interacts with to perform some task or descision. Currently <dialog> is not implemented in all major browsers. In order to provide a consistent experience across browsers our component omits the usage of the <dialog> element to avoid the implementation done by the browser.

For more information about the dialog element visit the W3C HTML 5.2 specification.
For more information about WAI-ARIA practices for the dialog element visit the W3C WAI-ARIA Authoring Practices 1.1

// HTML
<div role="dialog" class="nrk-dialog" aria-label="This is required!">
  <h2>Innhold i dialog</h2>
  <button>Godta</button>
  <button>Avslå</button>
</div>

// CODE
// Initialize a specific component or multiple components
window.coreComponents.dialog('.nrk-dialog');

// Open the dialog
window.coreComponents.dialog('.nrk-dialog').open();

// Close the dialog
window.coreComponents.dialog('.nrk-dialog').close();

FAQ

Why no grouped toggle?

Because, loosing scroll-position when expand / collapsing

Why not just use <details> instead of core-expand?

Despite having a native <details> element for expanding/collapsing content, there are several issues regarding browser support, styling, accessibility and polyfillis often conflict with other standards such as <dialog>

Why can I not just use plain progress elements?

Why not use web components?
