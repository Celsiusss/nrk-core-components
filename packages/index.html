@nrk/core-components

Getting started

core-components provides a easy, unstyled and accessbile layer of functionality. React/Preact (with preact-compat) or VanillaJS.

Use from NPM

- recommended 
npm install @nrk/core-toggle --save      # Components are individually installed to achieve individual versioning
import coreToggle from '@nrk/core-toggle'     // Vanilla JS
import Toggle from '@nrk/core-toggle/jsx' // ...or React/Preact compatible JSX

Use with Require.js

 
<script>
  require(['https://static.nrk.no/core-components/latest/core-toggle/core-toggle.min.js'], function(coreToggle) {
    // use coreToggle() here
  });
</script>

Use with script tag

- good for prototyping, avoid in production 
<script src="https://static.nrk.no/core-components/latest/core-toggle/core-toggle.min.js"></script>
<script>
  // use window.coreToggle() here
</script>

Toggle

Use core-toggle to hide/show or expand/collapse contents. Has to be a button (use nrk-unset or nrk-button). Should place related element as next adjacent sibling, but can be placed as next screen-reader-reachable element (not same as focusable).

Use with JS

<button>Closed example<button>
<div hidden>Content</div>
import coreToggle from '@nrk/core-toggle'

coreToggle(elements, {
  open: false,
  popup: false
})

Note: hidden attribute to avoid flash of unstyled content

Use with JSX

import Toggle from '@nrk/core-toggle/jsx'

<Toggle open={false} popup={false} onToggle={function(){}}>
  <Toggle.Button className="nrk-button">
    Use with JSX
  </Toggle.Button>
  <Toggle.Content className="nrk-drop">
    Content
  </Toggle.Content>
</Toggle>

FAQ

Why not role menu etc?

Not that kind of menu https://www.w3.org/TR/wai-aria-practices-1.1/examples/menubar/menubar-1/menubar-1.html

Why no hover dropdown?

Because consistent UI, better for accesibility and mobile

Why no grouped toggle?

Because, loosing scroll-position when collapsing. Can be acchieved with toggle-event.

Why no select tag?

Because, mostly styling, and should be avoided anyway: https://www.youtube.com/watch?v=CUkMCQR4TpY

Why not just use <details> instead of core-expand?

Despite having a native <details> element for expanding/collapsing content, there are several issues regarding browser support, styling, accessibility and polyfillis often conflict with other standards such as <dialog>

Why can I not just use plain progress elements?

UU. That's why.

Why not use web components?

They are not mature for production usage yet.
